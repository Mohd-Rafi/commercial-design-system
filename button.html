Buttons

Download More View Figma File
Description

Buttons can be used to show the user’s choice of options for actions and assign these to a clear hierarchy. The target page to be accessed or the action to be triggered is shown on the buttons by means of a label in Roboto Regular or else by an icon, or a combination of the two.

Buttons are either black or white. Buttons can be grouped.

Button Sizes

The size of buttons depends on the base font size used which can be adapted to the viewport width.

  • Small button at a base font size of 12 px
  • Medium sized button at a base font size of 16 px

The minimum width of primary and secondary buttons is defined as nine times the base font size. Thus, with a base font size of 16 px, the minimum button width is 9 × 16 px = 144 px.

Primary and secondary button group

The primary button emphasises a more important action as compared to the secondary button. Only one primary button is used per view, but this can be combined with several secondary buttons. The gap between the buttons is small (recommendation: 0.25 rem, i.e. one quarter of the basic type size).

Text Button

The text button is applied as a subtle call to action. The standard variant is a combination of text and arrow. It is possible to combine a text button with an icon or else omit an icon. If an additional hierarchy level is required for a text button, its variant can be applied in Roboto Bold.

System Icon Button

System icon buttons are used to trigger specific system interactions (e.g. save, print, download, etc.) System icons can be used in the two sizes available. The system icon button is transparent and should be used without a label.

Floating Button

The floating button is used for any action if it is to be highlighted in a level above the layout. It has a fixed position in the viewport. Depending on the context it can be hidden or shown, either automatically or through user interaction.

Stretched Primary Button

Like the primary button, the stretched button can be used for the most important function of a page or view. Unlike the primary button, it does not follow the L/R edge spacing rules and stretches the width of the viewport (or container) it occupies. All primary buttons in viewports below 440px are stretched.
